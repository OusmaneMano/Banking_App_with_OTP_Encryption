# Secure OTP Authentication System

## Overview

This project was developed as part of a master’s thesis focused on improving OTP-based authentication systems for mobile banking applications.

The main objective was to address geographical limitations affecting traditional SIM-based OTP delivery systems. In some regions or countries, users may experience difficulties receiving SMS OTPs, preventing access to their mobile banking applications.

To solve this problem, the project introduces a secondary application responsible for securely receiving OTPs generated by the backend server without relying entirely on traditional SIM card delivery.

---

## Problem Statement

Many mobile banking applications depend on SMS-based OTP authentication. However, OTP delivery may fail when:

* The user is outside the supported country or network region
* SMS services are delayed or unavailable
* SIM-based delivery systems experience connectivity issues

As a result, legitimate users may lose access to their banking applications.

---

## Proposed Solution

The system consists of:

### 1. Banking Application

The main mobile banking application where users authenticate using username and password credentials.

### 2. Java Backend Server

The backend server:

* Generates OTPs
* Encrypts OTP data using AES encryption
* Stores and validates OTPs
* Sends OTP information securely to the OTP receiver application

### 3. OTP Receiver Application

A secondary mobile application designed to receive OTPs generated by the backend system, reducing dependency on traditional SIM-based SMS delivery.

---

## Security Features

* AES encryption for OTP protection
* OTP generation and verification workflow
* Secure backend communication
* MySQL database integration

## Project Goal

The goal of this project is to improve accessibility and reliability in OTP-based authentication systems for mobile banking environments, especially in situations where traditional SMS delivery is limited by geographical or network restrictions.
